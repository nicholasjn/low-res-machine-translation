# Low-Resource Machine Translation for Batak Toba

This repository contains the code, data generation scripts, and evaluation results for our research on Low-Resource Machine Translation specifically focusing on **Batak Toba (BBC)**. 

We explore and compare the performance of two distinct architectures: **NLLB-600M** and **Qwen2.5-1.5B-Instruct**, utilizing a mix of real, monolingual, and synthetic datasets.

## 🚀 Models Investigated

1. **NLLB-200-distilled-600M**: A specialized model for low-resource translation.
2. **Qwen2.5-1.5B-Instruct**: A capable large language model adapted for translation tasks.

## 📊 Datasets

Given the extremely low-resource nature of Batak Toba, we utilize a combination of data sources and synthetic generation techniques:

* **Real Translation Data**: Sourced from the NusaX dataset.
* **Monolingual Data**: Sourced from Wikipedia-Batak and MADLAD-400.
* **Synthetic Data (ID/EN to BBC)**: Generated from Wikipedia-Batak and MADLAD-400 monolingual data using the `googletrans` API.
* **Synthetic Data (BBC to ID/EN)**: Generated by translating NLLB and OPUS English-Indonesian (EN<->ID) translation pairs using the `googletrans` API.

## 🧪 Experiment Setup

### 1. NLLB Full Fine-tuning
We performed full fine-tuning on the `NLLB-200-distilled-600M` model using various configurations of real and synthetic sentence pairs, as well as a multi-stage curriculum approach.

### 2. Qwen2.5-1.5B-Instruct Two-Step Training
* **Step 1 (Continued Pre-training)**: Train on monolingual Batak Toba data to adapt the model to the target language's syntax, vocabulary, and grammar.
* **Step 2 (Supervised Fine-Tuning - SFT)**: Fine-tune the adapted model on synthetic and real sentence pairs for the translation task.

## 📈 Evaluation Setup

* **Test Set**: Evaluated on the NusaX test data (400 rows).
* **Metrics**: We use **BLEU** and **chrF++** to evaluate translation quality for both ID -> BBC and BBC -> ID directions.
